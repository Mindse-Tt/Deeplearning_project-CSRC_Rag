# 执行路线与验证计划

这份文档不是项目介绍，而是执行蓝图。目标是回答三个问题：

- 先做什么
- 每一步产出什么
- 每一步怎么验证

## 1. 推荐顺序

推荐顺序不是“先上大模型”，而是：

1. 数据工程
2. 检索 baseline
3. 意图路由与系统协议
4. 处罚类型预测 baseline
5. RAG 生成与幻觉控制
6. Demo 和答辩包装

原因：

- 数据工程决定后面所有任务是否成立
- 检索是 RAG 的地基
- 意图路由决定系统能回答什么问题
- 预测模型是课程对微调/监督学习要求的核心支撑
- Demo 最后做效率最高

## 2. Phase 1：数据工程

### 目标

- 读通 Excel
- 构造事件级知识库文档
- 构造当事人级训练样本

### 产出

- `data/processed/event_corpus.jsonl`
- `data/processed/party_samples.jsonl`
- 数据概况统计

### 验证

- 样本数量与原始数据规模一致
- `EventID` 聚合逻辑正确
- 训练输入中不包含 `PunishmentMeasure`
- 文本字段清洗后可直接用于建模

## 3. Phase 2：检索 baseline

### 目标

- 验证知识库是否真的能检索到相似案例

### 产出

- BM25 baseline
- Dense baseline
- Hybrid baseline
- 可比较的离线指标表

### 验证

- `Recall@5`
- `MRR`
- `nDCG@10`
- 检索示例人工抽查

## 4. Phase 3：意图路由与回答协议

### 目标

- 确定系统支持哪些问题
- 确定每种问题对应哪种检索范围和回答模板

### 产出

- 意图配置
- 简单路由器
- 回答结构协议

### 验证

- 给定示例问题时，意图识别符合预期
- 不同意图能走到不同检索路径

## 5. Phase 4：处罚类型预测 baseline

### 目标

- 建立监督学习基线
- 满足课程对“微调前后对比”的要求

### 产出

- `PunishmentType` 多标签分类基线
- 特征组合实验
- 模型指标表

### 验证

- `Micro-F1`
- `Macro-F1`
- `Hamming Loss`
- `Subset Accuracy`

## 6. Phase 5：RAG 生成与幻觉控制

### 目标

- 基于检索证据生成解释性回答
- 控制回答超出证据范围

### 产出

- RAG 回答链路
- 引证格式
- 幻觉对比实验

### 验证

- 是否引用真实案例
- 是否引用真实法条
- 是否能说明推荐处罚的依据
- `无RAG / 有RAG / 有RAG+强约束` 三组对比

## 7. Phase 6：Demo

### 目标

- 形成答辩可展示闭环

### 建议页面

- 问答页
- 相似案例页
- 处罚推荐页
- 统计页

### 验证

- 输入真实案例描述后可稳定返回结果
- 页面能展示案例、法条、推荐和解释

## 8. 当前最应该先写的代码

先写这四个部分：

1. 原始 Excel 读取器
2. 事件级 / 当事人级样本构造器
3. 意图配置与路由器
4. 检索与训练配置

这样后面补 BM25、Dense、分类模型、RAG 模型都能直接接上去。

## 9. 当前已经完成到哪里

截至当前版本，以下步骤已经不再是计划，而是已经落地：
