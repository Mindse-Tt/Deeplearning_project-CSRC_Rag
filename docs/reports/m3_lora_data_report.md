# M3 · LoRA 训练数据构造报告

> 生成脚本：`scripts/build_rag_qa_train.py`（执行阶段）
> 数据源：`data/processed/event_corpus.jsonl`（4233 事件）
> 产出：`data/processed/rag_qa_{train,val,test}.jsonl`

## 1. 八类实际数量（目标 vs 实际）

| 类别 | 目标 | 实际 | 说明 |
|------|------|------|------|
| A | 1800 | 1800 | 案例检索（基于 activity / violation_types 派生 query） |
| B | 1200 | 1200 | 法条依据（基于 law 字段派生 query，引用法条标签） |
| C | 1000 | 1000 | 处罚推荐（PunishmentMeasure 已从 input 中剔除防泄漏） |
| D | 400 | 400 | 趋势统计（按 violation_types + declare_date 聚合） |
| E | 300 | 300 | 拒答（越界问题，固定拒答话术） |
| F | 300 | 300 | 问候/闲聊（固定能力介绍话术） |
| G | 200 | 200 | 多轮跟进（history 共指消解，引用上一轮 EventID） |
| H | 300 | 300 | 反幻觉负样本（虚构公司 / EID，必须回复『未检索到』或『无法回答』） |
| **总计** | 5360 | 5360 | — |

## 2. 切分统计（train / val / test）

| 类别 | train | val | test |
|------|-------|-----|------|
| A | 1440 | 180 | 180 |
| B | 960 | 120 | 120 |
| C | 800 | 100 | 100 |
| D | 320 | 40 | 40 |
| E | 240 | 30 | 30 |
| F | 240 | 30 | 30 |
| G | 160 | 20 | 20 |
| H | 240 | 30 | 30 |
| **合计** | 4400 | 550 | 550 |

切分比例：train=0.800，val=0.100，test=0.100。

切分规则：
- A/B/C/D：按事件 `declare_date` 时间三分（train=1994–2021，val=2022–2023，test=2024–2025）；同一 event_id 绝不跨 split。
- E/F/H：无事件绑定，按 80/10/10 随机切分（`seed=42`）。
- G：随机 80/10/10；每个 split 内仅选取该 split 对应时间窗口的事件做 seed，避免多轮样本的 event 泄漏。

各 split 中出现过的不同 event_id 数量：train=2126, val=372, test=382。

## 3. 四条验证断言结果

- ✅ PASS　同一 event_id 不跨 train/val/test
- ✅ PASS　A/B/C/D 类 100% 含 `[EventID=` 或 `[法条：`
- ✅ PASS　C 类证据块不含 PunishmentMeasure / 罚款金额 等泄漏词
- ✅ PASS　H 类 output 100% 含『未检索到』或『无法回答』
