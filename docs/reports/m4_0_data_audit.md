## 审计方法

对 `data/processed/rag_qa_train.jsonl` 的 4400 条训练样本,验证每条样本的 `event_id_source` 字段是否在 `input`(检索证据块)中出现 `[EventID=xxx]` 格式。E/F/H 三类没有 event_id_source(越界/问候/反幻觉),标记为"该类无需 ID 校验"。

## 为什么不需要"noisy retrieval"对照训练集

原计划是造一份用真实检索器输出作为证据的对照训练集,训一个"带噪声"的 LoRA 做对比实验。结论:**不划算**。

1. **oracle 已覆盖正面信号**:A-D 类学"证据→正确引用→答案"的干净映射,这正是 Ch4.2 要的"微调让模型学会可溯源回答"。
2. **H 类已覆盖负面信号**:240 条反幻觉样本里证据为空,教模型在没证据时说"未检索到" —— 这等价于"检索 Recall=0 场景",已经训过。
3. **G 类已覆盖多轮场景**:代词消解 + oracle 证据 = 多轮追问的干净监督。
4. **训练 ≠ 推理分布一致**:LoRA 训出的 Responder 推理时配 Validator 校验层,EventID 不在检索证据里就 reject,不靠"训练见过噪声"来防幻觉。

如果 M4.4 评估后发现**推理时检索失败的场景里模型还是硬编答案**,再回来造 noisy 训练集补强。先跑一遍 oracle 训练看真实效果,避免在假设上堆工作。
