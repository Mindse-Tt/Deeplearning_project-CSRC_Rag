## 类型 A · G0 基线幻觉(已被微调缓解)

> G3 已将幻觉数字率从 18% 压到 2.0%。A 类问题在 G0 时的表现,可作为答辩时"对比前后"的核心论据。

### A1 · 罚款金额虚构

- **查询**: 某投资者集中资金连续买卖某股票操纵其价格获利 1000 万元以上,预计会被如何处罚?
- **G0 输出**: "将被责令停业,并处以 **30 万元以上 60 万元以下**的罚款;构成犯罪的,依法追究刑事责任。"
- **问题**: `30 万元 / 60 万元` 是模型编造的精确数字,证监会实际罚款远高于此范围。
- **G3 缓解**: 改为保守引用法条,不给具体金额。

### A2 · 编造公司 + 人名

- **查询**: 2022 年证监会查处的董事长因内幕交易被罚款的案件有哪些?
- **G0 输出**: "1. 深圳市天马科技股份有限公司**董事长陈伟明**涉嫌内幕交易案。2. 上海市华联股份有限公司**朱永强**..."
- **问题**: 公司名与人名均为模型虚构,知识库中无对应记录,属于高危事实幻觉。
- **G3 缓解**: `参考案例见 [EventID=40123812],具体条款适用以公告原文为准。`

### A3 · 编造法院与审理过程

- **查询**: 湖北九有投资股份有限公司因虚构利润被证监会处罚的案件是什么?
- **G0 输出**: "中国证监会决定对其采取责令改正措施,并处以 **20 万元罚款**。该案件由**湖北省高级人民法院一审审理**,湖北省武汉市中级人民法院二审..."
- **问题**: 证监会行政处罚不走法院审理流程,"法院一审/二审"完全不合实际流程。

### A4 · 虚构减持规则

- **查询**: 控股股东在减持期间超比例减持上市公司股票且未及时公告,预计会被如何处罚?
- **G0 输出**: "将被中国证监会处以警告,并处以 **30 万元以上 60 万元以下**的罚款。"
- **问题**: 金额范围错误。

## 类型 B · G3 微调后仍失败(7 / 30)

> 这是微调模型的剩余 24.0% 未合规样本。其中 B4、B5 已被**推理后处理过滤器**修复(2026-04-24 新增),其余需要扩训练数据或改 prompt。

### B1 · 文书编号 ≠ EID 格式 ⚠️ **未修复**

- **gold_id**: gold_083
- **查询**: 湖北九有投资股份有限公司因虚构利润被证监会处罚的案件是什么?
- **G3 输出**: "...主要违反的法规为:**湖北证监局行政处罚决定书[2017]4号**;中国证监会湖北监管局行政处罚决定书[2017]5号;其他..."
- **问题**: 模型把文书编号 `[2017]4号` 当成了 EventID 格式,但 L7 Validator 只认 `[EventID=xxx]` 模式,所以格式合规判为 ✘。
- **修复方案**: 训练数据补充 100 条区分"文书编号"与"EventID"的负例 → 重训(26-30 min)。

### B2 · 跳过引用,直接用模板话术 ⚠️ **未修复**

- **gold_id**: gold_051
- **查询**: 夏传武因内幕交易被证监会行政处罚的案件详情?
- **G3 输出**: "参考历史相似案例,对夏传武因内幕交易类行为进行处罚。最终处罚程度需结合违规情节、主观故意、危害后果综合认定,具体细节以证监会公告为准。"
- **问题**: 完全没引用任何 EID。根因是检索层 top-5 未命中 gold,模型保守拒答。
- **修复方案**: 两条路径 —— ①检索器 Recall@5 提升;②生成层允许"即便未命中 gold,也要引用检索返回的 top-1 EID"。

### B3 · "列多条案例" 答得敷衍 ⚠️ **未修复**

- **gold_id**: gold_006
- **查询**: 2025 年以来上市公司因推迟披露且存在占用公司资产行为被处罚的案例有哪些?
- **G3 输出**: "根据检索证据,此类上市公司因推迟披露且存在占用公司资产行为的处罚案例见下表。**其中典型例举多于 5 条**,具体数量需参考公告原文..."
- **问题**: 答"多于 5 条"但不列具体 EID。训练数据里"列多个案例"的样本不足。
- **修复方案**: 补充 Class A 里输出含 2+ `[EventID=]` 的样本至少 500 条,当前仅 1440 条(均含 2 个 EID)。

### B4 · EID 分隔符错误 ✅ **已修复(2026-04-24 后处理)**

- **gold_id**: gold_063
- **查询**: 陈苏益(董事长)因虚假记载被证监会处罚的案件情况如何?
- **G3 原始输出**: `[EventID=40109363；EventID=401737；EventID=4010573]`(全角分号)
- **修复后**: `[EventID=40109363][EventID=401737][EventID=4010573]`(标准分隔)
- **实现**: `src/csrc_rag/response/responder.py` `_EID_SEMICOLON_RE` 正则在解码后自动规范化。

### B5 · Prompt 模板变量泄漏 ✅ **已修复(2026-04-24 后处理)**

- **gold_id**: gold_108
- **查询**: 姜涛推迟披露案援引了《证券法》哪一条?
- **G3 原始输出**:
  ```
  参考历史相似案例,对姜涛推迟披露类行为 most_like_thing_1='未按《公开发行证券的公司信息披露内容与格式准则第2号……';most_faq_affected_event_1=None;most_faq_result_1=违反了《证券法》第六十七条第一款...
  ```
- **问题**: 模型输出里混进了 Python 变量名 `most_like_thing_1=`, `most_faq_affected_event_1=None`,这些是训练数据构造时的中间模板占位符,未被清理就用于推理。
- **根因排查** (Scout Agent 2 调研):
  - 扫描 `data/processed/rag_qa_train.jsonl` 4,400 条源样本,**0 条**含这些变量名。
  - 说明污染来自**推理期模型自行产出的幻觉负例**,而非训练集。
- **修复后**: `参考历史相似案例,对姜涛推迟披露类行为`(变量名完全清除)
- **实现**: `src/csrc_rag/response/responder.py` `_VAR_LEAK_RE` + `_BARE_MARKER_RE` 正则在解码后过滤。

### B6–B7 · 其他零星失败

- 2 条样本因 LoRA 产出过于保守(完全不引用任何 EID),类似 B2。合并到 B2 统一处理。

## 类型 C · 检索瓶颈导致 EID 未命中 gold(17 / 30)

> G3 格式合规(76.0%)远高于 EID 命中(28.0%),差值主要来自这类:**LoRA 忠实地引用了检索 top-5 返回的 EID,但 top-5 没有命中 gold**。根因在检索层。

### C1 · Hard multi-hop 查询召回偏差

- **gold_id**: gold_008
- **查询**: 2023 年控股股东或高管操纵股价被罚的代表性案例有哪些?
- **G3 引用**: `[EventID=40148572][EventID=40137042]`
- **gold 要求**: 两条不同的 EventID
- **分析**: 检索层把 "操纵股价" 相关的案件召回了,但 gold 集挑的具体是另外两起。需要更细的元数据过滤(如按"控股股东"身份过滤)。

### C2 · 多约束拆分粒度不够

- **gold_id**: gold_002
- **查询**: 2022 年证监会查处的董事长因内幕交易被罚款的案件有哪些?
- **G3 引用**: `[EventID=40123812]`(单个)
- **gold 要求**: 多条
- **分析**: L2 多 sub-query 改写没有把 "2022"+"董事长"+"内幕交易"+"罚款" 四个约束合理拆分。

### C3 · 年份/身份元数据权重不足

- **gold_id**: gold_009
- **查询**: 2023 年上市公司控股股东因违规买卖本公司股票被处罚的案例有哪些?
- **G3 引用**: `[EventID=40136204]`
- **gold 要求**: 其他 EID
- **分析**: BM25 metadata 权重对 "2023 年" + "控股股东" 身份的加权还不够强。

### 共性修复建议

1. L2 Rewriter 的 sub-query 拆分增加 "年份 + 身份 + 行为" 三元组约束
2. BM25 metadata boost 调高 "身份"(如董事长 / 控股股东)字段权重
3. Reranker 做 CSRC 领域对比学习 LoRA(大工程,未来工作)

## 类型 D · 边界成功案例(应该在答辩时展示)

### D1 · L0 Topic Guard 拦截

- **查询**: 帮我预测明天股价
- **响应**: 硬拦截,返回拒答话术 `"不涉及股价预测 / 个股推荐 / 投资建议"`
- ✅ 行为符合预期

### D2 · out_of_scope 意图分类

- **查询**: 帮我写一首诗
- **响应**: Planner v2 判为 out_of_scope,走 fallback 文案
- ✅ 行为符合预期

### D3 · 经典成功案例

- **查询**: 谭光华因违规买卖股票被处罚的详情?
- **G3 输出**: `参考案例见 [EventID=40111147],具体条款适用以公告原文为准。`
- **gold 要求**: `[EventID=40111147]`
- ✅ 格式合规 + EID 命中 + 0 幻觉

### D4 · 新增成功案例(2026-04-24 修复)

- **查询**: 你是谁 / 你能干嘛 / 你是什么模型
- **原行为**: 被分类器分到 `chitchat`,走"超出我的专长"拒答文案 ❌
- **修复后**: 硬规则 `_GREETING_HARD_RULES` 覆盖分类器,固定走 greeting 的自我介绍文案 ✅
- **实现**: `src/csrc_rag/retrieval/engine.py` 新增 `_matches_greeting_rule` + `_planner_v2_early_exit` 里的 override

## 主动暴露的未修缺陷(答辩时讲)

以下 Bad Case 在本项目周期内**主动暴露,留作未来工作**,答辩时按"已识别 + 根因 + 路线图"三段式讲,这是成熟项目的诚实做法,非缺陷。

### B1 · 文书编号 ≠ EID 格式(未修)

- **现象**: `[2017]4号` 这种行政处罚决定书编号被模型误当作 EventID 格式
- **根因**: 训练数据未区分"文书编号"与"系统内部 EventID"两种不同的引用类型
- **修复路径**: 补 100 条区分样本 + 重训 LoRA (30 min)
- **为什么留到后续**: 数据工程成本 2-3h,边际收益 +5-10 pp 格式合规率,但不改变核心结论

### B3 · 列多条案例答得敷衍(未修)

- **现象**: 对"列举多起案例"类查询,LoRA 答"典型例举多于 5 条"但不列具体 EID
- **根因**: 训练集 Class A 里平均每条 output 只含 1 个 `[EventID=]`,未充分覆盖"列多条"模式
- **修复路径**: Class A 扩 500 条含 2+ EID 的样本 + 重训 (3-4 h)
- **为什么留到后续**: 对"幻觉率降 89%"这个核心结论没贡献,边际收益小

### C 类 · 检索层瓶颈(未修)

- **现象**: G3 格式合规率 76.0% 但 EID 命中 gold 仅 28.0%,中间 48 pp 差距 = 生成层引用了检索 top-5 内的 EID,但 top-5 没命中 gold
- **根因**: 检索层 Hybrid Recall@5 = 0.388 天花板,在 hard multi-hop query(多跳/多约束)上召回不足
- **修复路径(候选,由易到难)**:
  | 方案 | 工时 | 预期提升 | 可行性 |
  |---|---:|---:|---|
  | BM25 metadata 字段权重调优 | 2-4 h | +2-4 pp | 易,但可能过拟合 gold_130 |
  | 领域词典扩建 | 4-6 h | +1-2 pp | 易,收益小 |
  | Self-query LLM 拆分约束 | 8-12 h | +3-5 pp | 中,增加延迟 2-3s |
  | 换 bge-large-zh(1024 维) | 4 h | +1-3 pp | 易,显存 +1 GB |
  | **Reranker 领域对比学习 LoRA** | 16-24 h | +5-8 pp | 难,需 1000+ 对比样本 |
- **为什么留到后续**: 至少 16 h 的大工程,学期时间内不一定完成;业界 RAG 做完这些也仅 0.45-0.55 的 Recall@5,是"下一阶段研究"而非"本项目缺陷"

## 答辩要点

当老师问"你们系统有什么不好的?"时,按这个顺序讲:

1. **诚实承认类型 B 还有 24.0% 样本合规失败**(B1 B2 B3 未修)
2. **但强调 B4 B5 已通过推理后处理过滤修复**(展示 `_postprocess_answer` 前后对比)
3. **类型 C 的 72% EID 未命中**是检索层瓶颈,不是生成层错误 —— 这是我们 §4.1 + §7 局限里已经写明的
4. **类型 A 幻觉**是微调前,微调后 18% → 2.0% 已解决绝大部分

这样讲既诚实又能守住核心结论。
