CSRC-RAG

面向中国证监会违规处罚卷宗的检索增强问答系统（赛道 B 作品）。
基于 Qwen2.5-0.5B + QLoRA + BM25 + bge-small-zh + RRF 混合检索, 在 130 条 gold 集上对比微调前后效果, 把幻觉数字率从 18% 压到 2.0%。

Track B Qwen2.5-0.5B-Instruct + QLoRA BM25 + bge-small-zh-v1.5 + bge-reranker-v2-m3 RTX 2060 SUPER 8GB 4,233 events · 130 gold

核心成果（一句话）

LoRA 微调在 Qwen2.5-0.5B-Instruct 上达到了 断崖式 提升 —— 基线模型完全学不会 [EventID=xxx] 的引用格式（G0 / G1 / G2 三组全是 0.0%）， 微调后（G3）格式合规率直接拉到 76.0%，EventID 命中率 28.0%， 幻觉数字率从 18.0% 降到 2.0% （相对下降 -89%），推理延迟只增加 +0.7s（+7% 墙钟）。

四组生成对比（G0 → G3）

n=30 条 gold 分层抽样 (case_retrieval / law_grounding / sanction_recommendation)。 基座: Qwen2.5-0.5B-Instruct + 4-bit NF4 量化。 检索器: Hybrid (BM25 + bge-small-zh + RRF), top-5 证据。 采样: temperature=0.2, top_p=0.9, max_new_tokens=256。

L6 趋势聚合（M4.2）

趋势类问题（「2018-2023 年每年违规数?」「哪年最高?」）不是向量检索能准确回答的 — 我们把 trend_analysis 意图短路到一个 SQL-like 聚合层(TrendAggregator), 直接在 event_corpus 上做 groupby 计数。

MacBERT 辅线分类（M5）

当事人 → 处罚类型的多标签分类器 (7 类)。 基座 hfl/chinese-macbert-base, 轻量跑 1.2k 样本 × 1 epoch (CPU,留显存给 Qwen LoRA), 作为 sanction_recommendation 意图的辅助信号。

局限与诚实说明

  1. Base 是 0.5B 不是 1.5B: 本机 2060S 8GB 容不下 1.5B QLoRA, 0.5B 绝对能力弱于 1.5B。提供 Colab Notebook 作为 T4 上的 1.5B 重训方案。
  2. EID 命中 28% 看起来不高: 主要因为多 gold 命中要求严格 + 检索器在 hard multi-hop 上的天花板(M3e Hybrid Recall@5=0.388)。 实际格式合规 76.0%, LoRA 已经会正确写 EID, 命中是检索层瓶颈。
  3. 评测样本 30 条: 单轮次 30 条 95% CI 约 ±18%。后续可扩到 100 条。
  4. 幻觉检测只覆盖"数字类": 人名/公司名幻觉需人工标注, 见论文 §7。
