> 项目：CSRC-RAG · 证监会违规案例智能检索与问答系统
> 赛道：深度学习课程设计 · Track B
> 团队：许浩财 · 贾彤 · 戴一鑫 · 张彦扬 · 王怡菲
> 版本日期：2026-06

---

## 一、总体原则

本项目的**核心技术工作由团队成员独立设计与编写**，AI 工具仅在**非核心的辅助环节**（文档润色、样板代码、可视化脚本、排错建议等）起到提效作用。所有 AI 产生的内容均经过团队成员**逐项审阅、修改与验证**后才纳入项目；凡涉及方法设计、算法选择、实验控制、数据标注口径与最终结论判断，**均由人工拍板**。

一句话概括：

> **「系统怎么设计、为什么这么设计、实验怎么对照、结论是否成立」由人决定；
> AI 只是帮我们更快地把已经想清楚的东西写出来、画出来、查出错来。**

---

## 二、团队人工主导的核心工作

以下为本项目的**技术内核**，由团队成员独立完成设计与实现（对应代码已补充中文设计注释，可逐处追溯设计意图）：

| 模块 | 对应代码 | 人工主导内容 |
|------|----------|--------------|
| **七层 RAG 架构设计** | `src/csrc_rag/retrieval/engine.py` | 七层流水线（意图→改写→召回→精排→生成→趋势→校验）的整体编排与降级策略，均为团队自研设计 |
| **L1 意图分类** | `orchestration/intent_model.py`、`intents.py` | 7 类意图体系划分、TF-IDF + LogReg 选型、ML 优先+启发式兜底的路由逻辑 |
| **L2 查询改写** | `orchestration/rewriter.py`、`slot_filler.py` | 共指消解 + 同义词扩展 + 槽位抽取三段式设计、领域词典构建 |
| **L3 混合检索** | `retrieval/bm25.py`、`dense.py`、`hybrid.py`、`tokenizer.py` | BM25 + bge 稠密双路召回、RRF(k=60) 融合、jieba 领域分词的工程实现 |
| **L4 交叉编码器精排** | `retrieval/reranker.py` | bge-reranker 精排、权威度/严厉度加权、事件级去重聚合策略 |
| **L5 生成与 Prompt** | `response/responder.py`、`prompts.py` | System/Instruction 两层 prompt 设计、证据拼接格式、强制引证规则 |
| **L6 趋势聚合** | `orchestration/trend_aggregator.py` | 统计类问题绕过向量检索、改走 SQL-like groupby 的判断与实现 |
| **L7 引证校验** | `response/validators.py` | **8 条 YAML 校验规则**的设计（EID/法条/金额必须在证据内、越权措辞改写、失败降级话术等）——这是系统"会拒答、少幻觉"的关键防线 |
| **数据构造与标注口径** | `scripts/build_rag_qa_train.py` | A–H **八类任务**体系定义、按事件年份切分防泄漏、负样本（拒答/证据不足/反幻觉）的人工设计 |
| **微调方案与训练** | `scripts/train_qlora_qwen.py`、`training/` | LoRA r=16/α=32、fp16（适配 2060S Turing 不支持 bf16）、模型选型实验（Qwen vs Bloom × LoRA/QLoRA/Full FT）的设计与决策 |
| **评测体系** | `scripts/evaluate_generation_m4_4.py`、`evaluate_finetune_metrics.py` | 6 项指标定义、G0–G3 单变量递进消融的实验控制设计 |
| **金标数据人工标注** | `data/eval/` | 检索金标、意图评测集、趋势金标的人工标注与质量把控 |

> 上述模块的**设计决策、参数选择、实验结论**全部由团队成员负责；代码中的中文注释即为团队对各处设计意图的第一方说明。

## 三、AI 辅助的范围与边界

AI 工具在以下**非核心环节**提供了辅助，但所有产出均经人工审核修改：

| 环节 | AI 的作用 | 人工把关 |
|------|-----------|----------|
| 文档撰写 | 起草 README、策略文档、报告的初稿与排版 | 技术口径、数字、结论由人核对修正（见 README 数据已与代码逐项对齐） |
| 样板代码 | 生成 CLI 参数解析、日志、I/O 读写、HTTP handler 等重复性脚手架 | 由人整合进自研的核心逻辑并测试 |
| 可视化 | 协助编写论文配图、Mermaid 架构图的绘制脚本 | 图表数据与含义由人确认 |
| 代码排错 | 协助定位语法错误、依赖冲突、环境问题 | 修复方案由人判断与采纳 |
| 注释与整理 | 协助补充注释、统一代码风格 | 注释准确性由人逐处校对 |

**明确边界**：AI **没有**代替团队做以下任何一项——方法论设计、架构取舍、超参决策、实验对照设计、数据标注、结果解读与最终结论。
