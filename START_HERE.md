## 🎯 答辩核心三论点

### 论点 1️⃣: 微调是刚需，不是锦上添花

```
问题: G0/G1/G2 完全学不会 [EventID=xxx] 格式 (0% 合规率)
原因: <1B 小模型 zero-shot 无法遵循复杂格式约束  
解决: LoRA 微调使格式合规率直接拉到 76.0%

证据: docs/defense_talk.md 第二段有 3 条 G0→G3 改进示例
```

### 论点 2️⃣: 三层防线联合防幻觉

```
第一层 (检索约束)
  - 提供真实证据给模型
  - 幻觉率: 18% → 10% (-44%)
  
第二层 (微调约束)  
  - LoRA 微调学习数字约束
  - 幻觉率: 10% → 2% (-80%)
  
第三层 (规则约束)
  - 8 条 YAML 规则实时审计
  - 确保答案不超出证据范围

总体效果: 幻觉从 18% 压到 2%
剩余 2% 难以区分 (数据标注的人类主观性)
```

### 论点 3️⃣: EID 命中 28% 不低,是检索不是生成问题

```
观察: 格式合规 76.0% > EID 命中 28% (差 48 pp)

根因分析:
  - LoRA 已经会正确格式化答案
  - 但 top-5 证据中找不到 gold case
  - Hybrid Recall@5 = 0.388 是天花板 (业界也是 0.45-0.55)
  
结论:
  - 不是 LoRA 的问题,是检索层瓶颈
  - 这属于"下一阶段研究"而非"本项目缺陷"

改进方案:
  - 已在 FUTURE_WORK_ROADMAP.md 中列出 5 种方案
  - 8-24 小时可期待 +5-8 pp 收益
```

## 🔍 常见问题速答

### Q1: 为什么 EID 命中只有 28%?

A: G0/G1/G2 是 0%, G3 到 28% 是巨大进步。主要原因是检索瓶颈(Recall@5=0.388), 不是生成层 LoRA 的问题。详见答辩讲稿第三段。

### Q2: 为什么没修 B1/B3?

A: 这是**诚实的缺陷报告**。B1/B3 修复成本低(各 3.5 小时)但边际收益小(各 +5pp 以内),不改变核心结论。这是成熟项目的做法。

### Q3: 怎么保证不幻觉?

A: 三层防线:检索约束 → 微调约束 → 规则约束。实测幻觉率从 18% 降至 2.0%。详见答辩讲稿第三段。

### Q4: 10+ 秒延迟能接受吗?

A: 系统现状 10.54s(检索 3s + 排序 2s + LoRA 4s + 其他)。业界 RAG 通常 5-15s,我们在合理范围。合规查询允许 5-10s 延迟。
